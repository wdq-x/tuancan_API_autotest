# 食堂管理 - 订单模块接口文档（前端调试）

本文档面向前端开发，用于联调订单相关接口。包含完整 URL、请求方法、参数类型、必填说明及示例，便于直接调用与调试。

**基础信息：**

| 项目 | 说明 |
|------|------|
| **Base URL** | `/api/canteen-management/` |
| **认证方式** | 需登录，请求头携带 JWT：`Authorization: Bearer <token>` |
| **统一响应** | `{ "code": 0, "message": "success", "data": ..., "count": null, "next": null, "previous": null, "status": 200 }`；`code === 0` 表示成功 |

## 通用说明

### 分页参数（列表类接口）

| 参数名 | 类型 | 必填 | 默认 | 说明 |
|--------|------|------|------|------|
| page | int | 否 | 1 | 页码 |
| size | int | 否 | 20 | 每页条数（部分接口为 `page_size`，以具体接口为准） |

### 日期时间格式

- **Query 参数**：`create_date_start` / `create_date_end` 支持纯日期 `2025-02-13` 或日期时间 `2025-02-13 08:00:00`，按**中国时区**解析。
- **Body 中的 datetime**：建议 ISO 8601 或接口说明中的格式（如 `%Y-%m-%d %H:%M`）。

### 错误码与 HTTP 状态

- `code === 0`：成功。
- `4xx`：参数错误(400)、无权限(403)、资源不存在(404) 等，以 `message` 为准。
- `500`：服务器异常。

## 1. 订单状态列表

用于下拉、筛选、Tab 映射（待提交/已下单/已受理 等）。

| 项目 | 说明 |
|------|------|
| **URL** | `GET /api/canteen-management/order-status/` |
| **权限** | 已登录 |

### 请求参数（Query，均可选）

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| name | string | 否 | 状态名称过滤 |
| sign | string | 否 | 状态标记过滤 |
| active | boolean | 否 | 是否有效 |
| ordering | string | 否 | 排序，如 `sequence`、`id` |

### 响应 data

数组，每项：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 状态 id |
| name | string | 状态名称（如 待提交、已下单） |
| sign | string | 标记：to_pending, pending, consignment, stay_receiving, to_be_settled, complete, cancelled |
| sequence | int | 排序 |
| active | boolean | 是否有效 |

**说明**：接口会排除 `sign='cancelled'` 或 `name='已取消'` 的记录。

## 2. 配送商与品类（下单用）

根据当前用户当前食堂，查询该食堂对应单位下的**所有配送商及其可配送品类**。

| 项目 | 说明 |
|------|------|
| **URL** | `GET /api/canteen-management/order/distributor-cate/` |
| **权限** | 仅学校用户（token 中 `department_type === 'school'`），食堂从 token 的 `current_canteen.id` 取 |

### 请求参数

无（食堂从 token 解析）。

### 响应 data

数组，按配送商 id 排序，每项：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 配送商 id |
| name | string | 配送商名称（简称优先） |
| category_info | array | 品类列表，每项 `{ "id": 品类id, "name": "品类名称" }` |

## 5. 增加订单商品-可选商品列表

在「编辑订单」中「添加商品」时，根据订单 id 查询该订单对应配送商、单位下可选的商品列表（分页）。

| 项目 | 说明 |
|------|------|
| **URL** | `GET /api/canteen-management/order/aop/` |
| **权限** | 学校用户或 admin；非 admin 会校验订单属于当前用户所在学校 |

### 请求参数（Query）

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| order_id | int | **是** | 订单 id |
| cate_id | int | 否 | 品类 id，筛选该品类下商品 |
| name | string | 否 | 商品名称模糊搜索 |
| page | int | 否 | 页码，默认 1 |
| size | int | 否 | 每页条数，默认 20 |

### 响应

- 分页时：`data` 为分页结构（见通用分页），`data.results` 为数组。
- 每项：`id`（商品id）、`name`、`category_name`、`unit_name`、`specification`、`price`（询价单价，可能为空）。

## 6. 配送商商品列表（选品用）

与 Odoo 选品接口一致，按配送商、品类、关键字、预测周期等查商品；有合同时会多返回 `enquiry_set_price`、`procurement_permission` 等。

| 项目 | 说明 |
|------|------|
| **URL** | `GET /api/canteen-management/order/product/` |
| **权限** | 仅学校用户；食堂、用户从 token 解析 |

### 请求参数（Query）

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| distributor_id | int | 否 | 配送商 id |
| page | int | 否 | 页码，不传默认 1 |
| size | int | 否 | 每页条数，不传由后端默认 |
| cate_id | int | 否 | 品类 id |
| search | string | 否 | 商品关键字 |
| open_forecast | boolean | 否 | 是否按预测周期筛选 |
| open_cycle_start | string | 否 | 预测周期开始，YYYY-MM-DD |
| open_cycle_end | string | 否 | 预测周期结束，YYYY-MM-DD |

**注意**：`open_forecast=true` 时，`open_cycle_start`、`open_cycle_end` 必填，否则返回参数错误。

### 响应

- **无合同明细时**：仅返回 `data.dataList`（商品列表）及 `count`。
- **有合同明细时**：`data` 还包含 `enquiry_set_price`、`procurement_permission`、`first_distributor_id`、`first_distributor_ids` 等，具体以实际返回为准。

## 7. 采购计划提交

采购计划一键生成多笔订单（按配送商分组），异步创建，接口先返回成功。

| 项目 | 说明 |
|------|------|
| **URL** | `POST /api/canteen-management/order/purchase-plan/submit/` |
| **Content-Type** | `application/json` |
| **权限** | 仅学校用户；食堂从 token `current_canteen.id` 取 |

### 请求体（Body）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| warehouse_id | int | 否 | 入库库房 id，≥1 |
| products | array | **是** | 商品列表，至少一项，见下表 |
| default_except_time | string | 否 | 要求到货时间，格式 `YYYY-MM-DD HH:mm`，如 `2026-02-12 07:00`（中国时区） |

**products[] 元素：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| product_id | int/string | **是** | 商品 id（FsGoodsManagementAdvance.id） |
| purchase_quantity | number | **是** | 采购数量，≥0 |
| floor_quantity | string | 否 | 楼层数量，如 `1+0+0`，默认 `''` |
| distributor_id | int | **是** | 配送商 id，≥1 |
| distributor_price | string | 否 | 配送商价格，默认 `'0'` |

### 响应

- 成功：`code === 0`，`message` 如「采购计划提交成功！」；订单在后台异步创建，失败仅打日志。
- 失败：400，如缺少食材、未查询到食堂员工、`default_except_time` 格式错误等。

### 示例

```json
POST /api/canteen-management/order/purchase-plan/submit/
{
  "warehouse_id": 1,
  "default_except_time": "2026-02-13 07:00",
  "products": [
    { "product_id": 101, "purchase_quantity": 10, "distributor_id": 1, "floor_quantity": "1+0+0" },
    { "product_id": 102, "purchase_quantity": 5, "distributor_id": 1 }
  ]
}
```

## 8. 订单列表

分页、筛选、排序；学校用户仅能看自己负责食堂的订单。

| 项目 | 说明 |
|------|------|
| **URL** | `GET /api/canteen-management/order/` |
| **权限** | 已登录；学校用户按当前食堂过滤 |

### 请求参数（Query）

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| distributor_ids | string | 否 | 配送商 id 多选，逗号分隔，如 `1,2,3` |
| business_status_id | int | 否 | 业务状态 id（Tab：待提交/已下单等） |
| code | string | 否 | 订单号模糊 |
| consignor_tel | string | 否 | 送货人电话模糊 |
| product_names | string | 否 | 商品名称模糊（会按订单行匹配） |
| expected_time_start | string | 否 | 要求到货时间起始，datetime 格式 |
| expected_time_end | string | 否 | 要求到货时间结束，datetime 格式 |
| create_date_start | string | 否 | 下单时间起始，日期或日期时间（中国时区） |
| create_date_end | string | 否 | 下单时间结束，日期或日期时间（中国时区） |
| page | int | 否 | 页码 |
| size | int | 否 | 每页条数 |
| ordering | string | 否 | 排序，如 `-id`、`-create_date`、`expected_time`，多字段逗号分隔 |

### 响应（分页）

- `data`：分页结构，含 `results`、`count`、`next`、`previous` 等（以项目实际分页格式为准）。
- `data.results[]` 单条字段包括：id、code、sequence、category_names、product_names、expected_time、create_date、business_status_id、business_status_name、business_status_sign、state、remark、distributor_id、distributor_name、consignor_tel、canteen、canteen_name、custom_name、consignee_tel、receive_address、delivery_time、order_type、school、school_name、reconciliation_code、should_pay、actual_pay、service_pay、write_date、is_red 等。

## 12. 批量提交订单

待提交 → 已下单；所选订单须全部为「待提交」且已填「要求到货时间」。

| 项目 | 说明 |
|------|------|
| **URL** | `POST /api/canteen-management/order/submit/` |
| **Content-Type** | `application/json` |
| **权限** | 已登录；学校用户仅能提交自己负责食堂的订单 |

### 请求体（Body）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| order_ids | array[int] | **是** | 待提交的订单 id 列表，至少一个；须全部为 business_status.sign === 'to_pending' |

### 响应

- 成功：`code === 0`，`message` 如「提交成功」。
- 失败：400，如「请选择待提交的订单」「部分订单不存在或无权限操作」「批量提交的订单中有不是待提交的订单…」「请先填写要求到货时间！」等。

### 示例

```json
POST /api/canteen-management/order/submit/
{ "order_ids": [1, 2, 3] }
```

### 路径参数

## 14. 订单编辑/保存

更新要求到货时间、入库库房、备注、订单行（增删改数量/楼层量）。仅订单处于可编辑状态且订单行在可编辑状态时可改。

| 项目 | 说明 |
|------|------|
| **URL** | `PATCH /api/canteen-management/order/{id}/` 或 `PUT /api/canteen-management/order/{id}/` |
| **Content-Type** | `application/json` |
| **权限** | 已登录；学校用户仅能编辑自己负责食堂的订单 |

### 路径参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | int | **是** | 订单 id |

### 请求体（Body，仅传需更新字段）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| expected_time | string (datetime) | 否 | 要求到货时间 |
| warehouse_id | int | 否 | 入库库房 id，≥1 |
| remark | string | 否 | 订单备注 |
| order_lines | array | 否 | 订单行全量：有 id 为编辑，无 id 为新增，请求中未出现的已有行视为删除 |

**order_lines[] 元素：**

- **编辑已有行**：传 `id`（订单行 id），可选 `purchase_quantity`、`floor_quantity`。
- **新增行**：不传 `id`，传 `product_id`（商品 id）、`purchase_quantity`（必填），可选 `floor_quantity`。

**业务规则摘要：**

- 学校方在订单状态为 待收货/已收货/已结算/已取消 时不可编辑。
- 拣货中/已拣货完成（sale_mark 为 ing/end）时不可编辑。
- 订单行仅在订单 state 为 consignment、pending、cancelled 时可编辑。
- 保存后订单至少保留一条订单行，否则返回「订单商品不能为空」。

### 响应

- 成功：`code === 0`，`message` 如「保存成功」，`data` 为更新后的订单详情（与「订单详情」结构一致）。
- 失败：400，如状态不允许编辑、订单商品不能为空、存在商品未找到或不属于当前配送商等。

### 示例（仅更新备注）

```json
PATCH /api/canteen-management/order/123/
{ "remark": "请尽早配送" }
```

### 示例（更新订单行：改数量 + 新增一行）

```json
PATCH /api/canteen-management/order/123/
{
  "expected_time": "2026-02-14T00:00:00Z",
  "order_lines": [
    { "id": 1001, "purchase_quantity": 20, "floor_quantity": "1+1+0" },
    { "product_id": 205, "purchase_quantity": 5, "floor_quantity": "" }
  ]
}
```
说明：订单行 id 1001 保留并更新数量与楼层量；新增一条商品 205；其他原有行若未出现在 `order_lines` 中会被删除（软删）。
